Materi MySQL
MySQL adalah sistem manajemen basis data relasional (RDBMS) yang bersifat open-source dan menggunakan bahasa SQL (Structured Query Language) untuk mengelola data. MySQL banyak digunakan dalam aplikasi web dan dikembangkan oleh Oracle Corporation. Berikut adalah beberapa materi dasar untuk memahami dan menggunakan MySQL:
  • Create, Alter, dan Drop Table pada MySQL

    • CREATE TABLE

    Perintah CREATE TABLE digunakan untuk membuat tabel baru di dalam database MySQL. Dengan perintah ini, kamu dapat menentukan nama tabel, kolom-kolom yang ingin dibuat, tipe data tiap kolom, serta batasan seperti PRIMARY KEY atau NOT NULL.

    Contoh Sintaks: 
    CREATE TABLE penjualan (
              id INT PRIMARY KEY,
              nama_produk VARCHAR(100),
              harga_produk INT,
              keterangan VARCHAR(100)
            );
    • id: kolom bertipe integer yang akan bertambah otomatis setiap ada data baru.
    • nama_produk: teks maksimal 100 karakter dan tidak boleh kosong (NOT NULL).
    • harga_produk: bertipe integer, menyimpan harga barang.
    • keterangan: teks singkat berisi deskripsi produk.

    Catatan: Setiap tabel sebaiknya memiliki kolom PRIMARY KEY sebagai penanda unik untuk tiap baris data.


    Tambahan: Tipe Data dalam MySQL

    Dalam membuat tabel menggunakan CREATE TABLE, kamu perlu memilih tipe data yang sesuai untuk setiap kolom. Tipe data menentukan jenis nilai yang dapat disimpan di kolom tersebut.

    • INT – digunakan untuk menyimpan angka bulat (misalnya: 100, -25, 5000).
    • VARCHAR(n) – digunakan untuk teks dengan panjang maksimal n karakter (misalnya: VARCHAR(100)).
    • TEXT – digunakan untuk teks yang lebih panjang, seperti deskripsi produk atau catatan.
    • DATE – digunakan untuk menyimpan tanggal dalam format YYYY-MM-DD.
    • DECIMAL(p, q) – digunakan untuk angka desimal dengan ketelitian tertentu (misalnya: DECIMAL(10,2) untuk nilai 99999999.99).
    • BOOLEAN – menyimpan nilai logika, yaitu TRUE (1) atau FALSE (0).
    Tips: Pilih tipe data yang sesuai agar penyimpanan data lebih efisien dan proses query menjadi lebih cepat.

    ALTER TABLE

    Perintah ALTER TABLE digunakan untuk mengubah struktur tabel yang sudah ada, seperti menambah kolom baru, mengubah tipe data, mengganti nama kolom, atau menghapus kolom.

    Contoh Menambah Kolom: 
    ALTER TABLE penjualan
              ADD COLUMN stok INT AFTER harga_produk;
    Contoh Mengubah Nama Kolom: 
    ALTER TABLE penjualan
            CHANGE nama_produk nama_barang VARCHAR(100);
    Contoh Menghapus Kolom: 
    ALTER TABLE penjualan
            DROP COLUMN keterangan;

    Contoh: Berikut ini hasil ALTER TABLE dengan menambahkan kolom stok setelah kolom harga_produk

    Catatan: Gunakan ALTER TABLE dengan hati-hati karena perubahan struktur tabel bisa mempengaruhi data dan relasi antar tabel.

    DROP TABLE

    Perintah DROP TABLE digunakan untuk menghapus tabel beserta seluruh data di dalamnya dari database. Setelah dijalankan, tabel tidak bisa dikembalikan kecuali kamu memiliki cadangan data.

    Contoh Sintaks: 
    DROP TABLE penjualan;

    Jika kamu hanya ingin menghapus seluruh isi tabel tanpa menghapus strukturnya, gunakan TRUNCATE TABLE: 

    TRUNCATE TABLE penjualan;

    Contoh: Berikut ini hasil dariDROP TABLE

    Catatan: Gunakan DROP TABLE atau TRUNCATE hanya jika kamu benar-benar yakin, karena data yang sudah terhapus tidak dapat dikembalikan.

  • Manipulasi data CRUD pada MySQL

    • C (CREATE) - Memasukkan data

    Setelah membuat tabel, kita bisa menambahkan data dengan perintah INSERT INTO. Terdapat dua format umum:

    1. Menyebut semua kolom secara berurutan: 
    INSERT INTO penjualan VALUES ('1', 'iPhone 11 Promag', 17000000, 'Second');
    2. Menyebut kolom secara eksplisit: 
    INSERT INTO penjualan (id, nama_produk, harga_produk, keterangan)
              VALUES (1, 'iPhone 11 Promag', 17000000, 'Second');
    - Gunakan NULL untuk kolom yang AUTO_INCREMENT jika tidak menyebutkannya langsung. - Untuk banyak data, gunakan: 
    INSERT INTO penjualan (id, nama_produk, harga_produk, keterangan) VALUES
              (1, 'iPhone 11 Promag', 17000000, 'Second'),
              (2, 'Oddo Reno 5', 1700000, 'New'),
              (3, 'Samsung S25 Ultramilk', 20000000, 'Kredit'),
              (4, 'Semen Gresik', 100000, 'Preloved');

    Contoh: Berikut ini contoh dariINSERT INTO untuk memasukkan banyak data

    R (READ) - Menampilkan data

    Untuk menampilkan data dari tabel, gunakan perintah SELECT. Perintah ini berfungsi untuk membaca (read) data dari tabel.

    1. Menampilkan semua data dari tabel: 
    SELECT * FROM penjualan;
    2. Menampilkan kolom tertentu saja: 
    SELECT nama_produk, harga_produk FROM penjualan;
    3. Menampilkan data dengan kondisi tertentu: 
    SELECT * FROM penjualan WHERE keterangan = 'Second';
    4. Mengurutkan data berdasarkan harga: 
    SELECT * FROM penjualan ORDER BY harga_produk DESC;

    Contoh: Menampilkan data dengan kondisi tertentu (menampilkan data yang keterangannya second)

    U (UPDATE) - Mengubah data

    Untuk mengubah data di tabel, gunakan perintah UPDATE. Pastikan selalu menambahkan klausa WHERE agar data yang diubah spesifik.

    1. Mengubah satu kolom: 
    UPDATE penjualan SET harga_produk = 18000000 WHERE id = 1;
    2. Mengubah lebih dari satu kolom: 
    UPDATE penjualan
              SET harga_produk = 17500000, keterangan = 'New'
              WHERE id = 1;
    3. Mengubah data berdasarkan kondisi tertentu: 
    UPDATE penjualan SET keterangan = 'Diskon'
            WHERE harga_produk > 10000000;

    Contoh: Mengubah harga produk “Oddo Reno 5” menjadi Rp 1.800.000: 

    UPDATE penjualan SET harga_produk = 1800000 WHERE id = 2;

    D (DELETE) - Menghapus data

    Untuk menghapus data dari tabel, gunakan perintah DELETE. Gunakan klausa WHERE untuk menghindari penghapusan seluruh data.

    1. Menghapus satu baris data berdasarkan ID: 
    DELETE FROM penjualan WHERE id = 4;
    2. Menghapus berdasarkan kondisi tertentu: 
    DELETE FROM penjualan WHERE keterangan = 'Preloved';
    3. Menghapus seluruh data di tabel: 
    DELETE FROM penjualan;
    (Gunakan dengan hati-hati!)

    Contoh: Menghapus produk dengan ID = 3: 

    DELETE FROM penjualan WHERE id = 3;
  • Filtering data pada MySQL

    • FILTERING DATA

    Perintah SELECT di MySQL digunakan untuk menampilkan data dari tabel. Agar hasilnya lebih spesifik, kamu dapat menambahkan klausa WHERE untuk memfilter atau menyaring data sesuai kondisi yang diinginkan.

    Misalnya, kita memiliki tabel bernama penjualan dengan kolom: id, nama_produk, harga_produk, dan keterangan.

    Contoh Menampilkan Semua Data: 
    SELECT * FROM penjualan;
    Contoh Menampilkan Data dengan Kondisi Tertentu: 
    SELECT * FROM penjualan
            WHERE harga_produk > 100000;

    Perintah di atas menampilkan semua produk yang memiliki harga lebih dari 100.000.

    Contoh Menampilkan Produk Berdasarkan Nama: 
    SELECT * FROM penjualan
            WHERE nama_produk = 'Laptop';

    Perintah tersebut hanya akan menampilkan data produk dengan nama Laptop.

    Contoh Menggunakan Operator Logika: 
    SELECT * FROM penjualan
              WHERE harga_produk >= 1000000 AND keterangan = 'Elektronik';

    Hasilnya menampilkan semua produk yang harganya di atas atau sama dengan 1.000.000 dan memiliki keterangan "Elektronik".

    Contoh Menggunakan Operator LIKE (Pencarian Pola Teks): 
    SELECT * FROM penjualan
              WHERE nama_produk LIKE '%phone%';

    Operator LIKE digunakan untuk mencari pola teks. Tanda persen (%) berfungsi sebagai wildcard. Contoh di atas akan menampilkan semua produk yang memiliki kata "phone" di dalam namanya (misalnya: Smartphone, Headphone).

    Contoh Mengurutkan Hasil Filter: 
    SELECT * FROM penjualan
              WHERE harga_produk > 500000
              ORDER BY harga_produk DESC;

    Perintah ORDER BY digunakan untuk mengurutkan hasil. Tambahkan DESC untuk urutan dari yang terbesar ke terkecil, atau ASC untuk urutan dari yang terkecil ke terbesar. Gambar di bawah ini menunjukkan data yang memuat harga_produk yang diurutkan dari yang terbesar hingga terkecil

    Catatan: Filtering data sangat berguna untuk mencari informasi spesifik dalam jumlah data yang besar. Gunakan kombinasi WHERE, AND, OR, dan LIKE untuk menghasilkan hasil pencarian yang akurat dan efisien.

  • Fungsi Agregat pada MySQL

    • Fungsi Agregat digunakan untuk melakukan perhitungan terhadap sekumpulan data dalam tabel. Biasanya digunakan bersama perintah SELECT untuk menampilkan hasil perhitungan seperti jumlah data, rata-rata, nilai maksimum, minimum, dan sebagainya.

    Fungsi-fungsi agregat berguna ketika kita ingin mendapatkan ringkasan informasi dari data yang besar tanpa harus melihat seluruh baris satu per satu.

    🔹 Macam-macam Fungsi Agregat:
    • COUNT() → Menghitung jumlah baris data.
    • SUM() → Menjumlahkan nilai dari suatu kolom numerik.
    • AVG() → Menghitung rata-rata nilai.
    • MAX() → Menampilkan nilai terbesar.
    • MIN() → Menampilkan nilai terkecil.
    🔹 Contoh Penggunaan:
    SELECT COUNT(*) AS jumlah_data FROM penjualan;

    Query di atas menghitung jumlah seluruh baris data dalam tabel penjualan. 

    SELECT SUM(harga) AS total_harga FROM penjualan;

    Query ini menghitung total harga seluruh produk dari kolom harga dalam tabel penjualan.

    Gambar di atas menampilkan total keseluruhan harga_produk jika dijumlahkan

    Fungsi agregat juga bisa digabungkan dengan klausa GROUP BY untuk menghitung data berdasarkan kelompok tertentu. 

    
              SELECT kategori, SUM(harga) AS total_harga
              FROM penjualan
              GROUP BY kategori;

    Query tersebut akan menjumlahkan harga berdasarkan setiap kategori barang dalam tabel penjualan.

  • Subquery pada MySQL
    • SUBQUERY adalah query di dalam query lainnya. Biasanya digunakan di bagian SELECT, FROM, atau WHERE.

    Contoh: Menampilkan produk dengan harga terendah 
    
            SELECT * 
            FROM penjualan
            WHERE harga_produk = (SELECT MIN(harga_produk) FROM penjualan);

    Data yang ditampilkan adalah produk yang memiliki harga paling kecil pada tabel penjualan

    Jenis SUBQUERY: - Dalam SELECT: untuk nilai tambahan. - Dalam WHERE: sebagai filter. - Dalam FROM: sebagai tabel sementara (alias).

  • NOT dan LIMIT pada MySQL

    • NOT

    Operator NOT digunakan untuk menegasikan (membalik) kondisi dalam query. Artinya, NOT akan menampilkan data yang tidak memenuhi kriteria tertentu. Operator ini sering dipakai untuk memfilter data yang tidak termasuk dalam suatu kategori.

    Contoh: 
    
                SELECT * FROM penjualan
                WHERE NOT keterangan = 'Diskon';

    Query di atas akan menampilkan semua data produk dari tabel penjualan yang tidak memiliki keterangan "Diskon". 

    
              SELECT * FROM penjualan
              WHERE nama_produk NOT LIKE 'A%';

    Query ini menampilkan produk yang namanya tidak diawali huruf "A". 

    
              SELECT * FROM penjualan
              WHERE harga_produk NOT BETWEEN 100000 AND 2000000;

    Query ini menampilkan produk dengan harga di luar rentang 100.000 hingga 2.000.000. Berikut ini hasil dari query di atas

    Data yang ditampilkan adalah produk yang harganya bukan di antara 100.000 dan 2.000.000


    LIMIT

    Perintah LIMIT digunakan untuk membatasi jumlah baris yang ditampilkan dari hasil query. Fitur ini sangat berguna ketika kamu hanya ingin melihat sebagian data atau menerapkan pagination dalam aplikasi.

    Contoh: 
    
              SELECT * FROM penjualan
              LIMIT 3;

    Query ini hanya akan menampilkan 3 baris pertama dari tabel penjualan. 

    
              SELECT * FROM penjualan
              LIMIT 2, 3;

    Query ini akan melewati 2 baris pertama dan menampilkan 3 baris berikutnya (baris ke-3 sampai ke-5). 

    
              SELECT * FROM penjualan
              WHERE harga_produk > 50000
              LIMIT 2;

    Query ini menampilkan 2 produk pertama dengan harga di atas 50.000. Lihat gambar di bawah ini

    Gambar di atas menampilkan 2 produk pertama yang harganya di atas 50.000

    Catatan: Gunakan LIMIT saat menampilkan data dalam jumlah besar agar query lebih efisien dan tampilan data lebih mudah dibaca.
